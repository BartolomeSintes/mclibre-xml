Principales normas y recomendaciones XML

En este apartado se comentan las últimas versiones de las principales recomendaciones. Las versiones obsoletas y las notas de los grupos de trabajo se comentan en la página Versiones obsoletas.

Actividad XML en el W3C: https://www.w3.org/XML/

XML Core

Grupo de trabajo XML Core en el W3C: Homepage - Charter: 2013-2015 - 2015-2017

El grupo de trabajo XML Core se cerró en agosto de 2016.

El grupo de trabajo XML Core publicó las siguientes recomendaciones:

  • XML (Extensible Markup Language = Lenguaje de Marcas Extensible)

    • agosto de 2006: XML 1.1 (2ª edición), editada por Tim Bray y otros.
    • noviembre de 2008: XML 1.0 (5ª edición), editada por Tim Bray y otros.

    La recomendación XML 1.1 modifica el concepto de documento bien-formado y se adapta a las modificaciones ocurridas en el juego de caracteres Unicode. Las diferencias entre XML 1.0 y XML 1.1 no son demasiado importantes, por lo que la versión XML 1.0 sigue siendo la más utilizada.

    Tim Bray publicó una versión comentada de la primera versión recomendación XML 1.0.

  • Namespaces in XML

    Los espacios de nombres es el mecanismo que permite que diferentes lenguajes puedan compartir etiquetas sin ambigüedad.

    • agosto de 2006: Namespaces in XML 1.1 (2ª edición), editada por Tim Bray y otros.
    • diciembre de 2009: Namespaces in XML 1.0 (3ª edición), editada por Tim Bray y otros.
  • XML Schema

    Las esquemas XML son una evolución de las DTD.

    • octubre de 2004: XML Schema Part 0: Primer (2ª edición), editada por David C. Fallside y Priscilla Walmsley.
    • octubre de 2004: XML Schema Part 1: Structures (2ª edición), editada por Henry S. Thompson y otros.
    • octubre de 2004: XML Schema Part 2: Datatypes (2ª edición), editada por Paul V. Biron y Ashok Malhotra.
    • abril de 2012: W3C XML Schema Definition Language (XSD) 1.1 Part 1: Structures, editada por Shudi (Sandy) Gao y otros.
    • abril de 2012: W3C XML Schema Definition Language (XSD) 1.1 Part 2: Datatypes, editada por David Peterson y otros.
  • Otros
    • febrero de 2004: XML Information Set (2ª edición), editada por John Cowan y Richard Tobin.
    • septiembre de 2005: xml:id 1.0, editada por Jonathan Marsh y otros.
    • noviembre de 2006: XML Inclusions (XInclude) 1.0 (2ª edición), editada por Jonathan Marsh y David Orchard.
    • mayo de 2008: Canonical XML 1.1, editada por John Boyer y Glenn Marcy.
    • enero de 2009: XML Base (2ª ed), editada por Jonathan Marsh y Richard Tobin.
    • octubre de 2010: Associating Style Sheets with XML documents 1.0 (2ª edición), editada por James Clark.

Notas:

  • Las recomendaciones sobre XML Base fueron elaboradas por el grupo de trabajo XML Linking, cuya actividad cesó en 2005. Sus recomendaciones fueron mantenidas desde entonces por el grupo de trabajo XML Core.
  • La recomendaciones sobre XML Schema fueron elaboradas por el grupo de trabajo XML Schema, cuya actividad cesó en 2013 (último charter, hasta enero de 2013). Sus recomendaciones fueron mantenidas desde entonces por el grupo de trabajo XML Core.

XSL (eXtensible Style Language)

XSL es un conjunto de recomendaciones que definen la transformación y presentación de un documento XML.

Grupo de trabajo XSL en el W3C: Homepage - Charter: 2013-2015 - 2015-2017 (extendido hasta abril de 2018)

El grupo de trabajo XSL se cerró en octubre de 2018.

El grupo de trabajo XSL publicó las siguientes recomendaciones:

  • XPath
    • noviembre de 1999: XML Path Language 1.0, editada por James Clark y Steve DeRose.
    • enero de 2007: XML Path Language 2.0, editada por Anders Berglund y otros.
    • diciembre de 2010: XML Path Language 2.0 (2ª edición), editada por Anders Berglund y otros.
    • abril de 2014: XML Path Language (XPath) 3.0, editada por Jonathan Robie y otros.
    • marzo de 2017: XML Path Language (XPath) 3.1, editada por Jonathan Robie y otros.
  • XSLT (XSL Transformations)
    • noviembre de 1999: XSL Transformations 1.0, editada por James Clark.
    • enero de 2007: XSL Transformations 2.0, editada por Michael Kay.
    • junio de 2017: XSL Transformations 3.0, editada por Michael Kay.
    • marzo de 2021: XSL Transformations 2.0 (2ª edición), editada por Michael Kay.
  • XSL/FO (XSL Formatting Objects)
    • octubre de 2001: Extensible Stylesheet Language (XSL) 1.0, editada por Sharon Adler y otros.
    • diciembre de 2006: Extensible Stylesheet Language (XSL) 1.1, editada por Anders Berglund.

Una segunda versión XSL/FO (XSL Formatting Objects) se abandonó cuando todavía estaba en elaboración:

  • Extensible Stylesheet Language (XSL) Requirements Version 2.0, en estado de borrador desde 2008.
  • Extensible Stylesheet Language (XSL) 2.0, en estado de borrador desde 2012.

XQuery (XML Query)

El objetivo de XQuery es proporcionar herramientas para realizar consultas sobre documentos XML.

Grupo de trabajo XQuery en el W3C: Homepage - Charter: 2013-2015 - 2015-2017 (extendido hasta abril de 2018)

El grupo de trabajo XQuery se cerró en octubre de 2018.

El grupo de trabajo XQuery publicó las siguientes recomendaciones:

  • diciembre de 2010: XQuery 1.0: An XML Query Language (2ª edición), editada por Scott Boag y otros.
  • diciembre de 2010: XML Syntax for XQuery 1.0 (XQueryX) (2ª edición), editada por Jim Melton y Subramanian Muralidhar.
  • marzo de 2011: XQuery Update Facility 1.0, editada por Jonathan Robie y otros.
  • abril de 2014: XQuery 3.0: An XML Query Language, editada por Jonathan Robie y otros.
  • abril de 2014: XQueryX 3.0, editada por Jim Melton.
  • marzo de 2017: XQuery 3.1: An XML Query Language, editada por Jonathan Robie y otros.
  • marzo de 2017: XQueryX 3.1, editada por Jim Melton y Josh Spiegel.

Nota: Después de XQuery 1.0 se publicó XQuery 3.0, es decir, no existieron ni XQuery 1.1 ni XQuery 2.0. Probablemente la razón fue utilizar el mismo número de versión que XPath 3.0, publicado en la misma fecha (abril de 2014).
