Principales normas y recomendaciones XML

En este apartado se comentan las últimas versiones de las principales recomendaciones. Las versiones obsoletas de estas recomendaciones se comentan en la página Versiones obsoletas.

Actividad XML en el W3C: https://www.w3.org/XML/

XML Core

Grupo de trabajo XML Core en el W3C: Homepage - Plan de trabajo (2015-2017) - Plan de trabajo (2013-2015)

El grupo de trabajo XML Security se cerró en agosto de 2016. El grupo de trabajo XML Core publicó las siguientes recomendaciones:

  • XML (Extensible Markup Language = Lenguaje de Marcas Extensible)
    • noviembre de 2008: XML 1.0 (5º edición), editada por Tim Bray y otros
    • agosto de 2006: XML 1.1 (2º edición), editada por Tim Bray y otros

    La recomendación XML 1.1 modifica el concepto de documento bien-formado y se adapta a las modificaciones ocurridas en el juego de caracteres Unicode. Las diferencias entre XML 1.0 y XML 1.1 no son demasiado importantes, por lo que la versión XML 1.0 sigue siendo la más utilizada.

    Tim Bray publicó una versión comentada de la primera versión recomendación XML 1.0.

  • Namespaces in XML

    Los espacios de nombres es el mecanismo que permite que diferentes lenguajes puedan compartir etiquetas sin ambigüedad.

    • diciembre de 2009: Namespaces in XML 1.0 (3º edición), editada por Tim Bray y otros
    • agosto de 2006: Namespaces in XML 1.1 (2º edición), editada por Tim Bray y otros
  • XML Inclusions (XInclude)
    • noviembre de 2006: XML Inclusions (XInclude) 1.0 (2º edición), editada por Jonathan Marsh y David Orchard
  • XML Information Set
    • febrero de 2004: XML Information Set (2º edición), editada por John Cowan y Richard Tobin
  • xml:id
    • septiembre de 2005: xml:id 1.0, editada por Jonathan Marsh y otros
  • XML Base
    • enero de 2009: XML Base (2º ed), editada por Jonathan Marsh y Richard Tobin
  • Hojas de estilo en XML
    • octubre de 2010: Associating Style Sheets with XML documents 1.0 (2º edición), editada por James Clark
  • XML Linking Language (XLink)
    • mayo de 2010: XML Linking Language (XLink) 1.1, editada por Steve DeRose y otros
  • XPointer
    • marzo de 2003: XPointer Framework, editada por Paul Grosso y otros
    • marzo de 2003: XPointer element() Scheme, editada por Paul Grosso y otros
    • marzo de 2003: XPointer xmlns() Scheme, editada por Steven J. DeRose y otros
  • XML Schema

    Las esquemas XML son una evolución de las DTD.

    • octubre de 2004: Esquema XML Parte 0: Fundamentos (2º edición), editada por David C. Fallside y Priscilla Walmsley
    • octubre de 2004: Esquema XML Parte 1: Estructuras (2º edición), editada por Henry S. Thompson y otros
    • octubre de 2004: Esquema XML Parte 2: Tipos de datos (2º edición), editada por Paul V. Biron y Ashok Malhotra
    • abril de 2012: Esquema XML 1.1 Parte 1: Estructuras, editada por Shudi (Sandy) Gao y otros
    • abril de 2012: Esquema XML 1.1 Parte 2: Tipos de datos, editada por David Peterson y otros
  • Otros
    • octubre de 2012: Associating Schemas with XML documents 1.0 (3º edición), editada por Paul Grosso y Jirka Kosek
    • mayo de 2008: Canonical XML 1.1, editada por John Boyer y Glenn Marcy

El grupo de trabajo XML Core publicó las siguientes notas del grupo de trabajo:

  • julio de 2016: XML Inclusions (XInclude) 1.1, editada por Jonathan Marsh y otros

Al cerrarse el grupo de trabajo, dos recomendaciones quedaron como borradores que ni siquiera se publicaron como notas del grupo de trabajo:

  • XML Schema Definition Language (XSD): Component Designators, editada por Mary Holstege y Asir S. Vedamuthu, en estado de borrador desde 2010
  • XML Normalization, editada por John Boyer y otros, en estado de borrador desde 2013

Notas:

  • Las recomendaciones sobre XLink, XML Base y XPointer fueron elaboradas por el grupo de trabajo XML Linking, cuya actividad cesó en 2005 (último plan de trabajo, hasta diciembre de 2002). Sus recomendaciones son mantenidas desde entonces por el grupo de trabajo XML Core.
  • La recomendaciones sobre XML Schema fueron elaboradas por el grupo de trabajo XML Schema, cuya actividad cesó en 2013 (último plan de trabajo, hasta enero de 2013). Sus recomendaciones son mantenidas desde entonces por el grupo de trabajo XML Core.
  • La norma Associating Schemas with XML documents 1.0 (3º edición) no está publicada como recomendación del W3C, sino como nota del grupo de trabajo, ya que es idéntica a la norma ISO/IEC 19757-11. El subcomité ISO/IEC JTC 1/SC 34 se encarga de la norma ISO 19757 (DSDL: Document Schema Definition Languages) que trata de varios lenguajes de definición de documentos (DTD, Schema, RELAX NG, Schematron, etc.). Algunas partes de esta norma se encuentran disponibles.

XSL (eXtensible Style Language)

XSL es un conjunto de recomendaciones que definen la transformación y presentación de un documento XML.

Grupo de trabajo XSL en el W3C: Homepage - Plan de trabajo (2015-2017) (extendido hasta abril de 2018) - Plan de trabajo (2013-2015)

Actualmente (abril de 2018) el grupo XSL no está cerrado, pero no se plantea publicar nuevas recomendaciones.

El grupo de trabajo XSL ha publicado las siguientes recomendaciones:

  • XPath
    • noviembre de 1999: XML Path Language 1.0, editada por James Clark y Steve DeRose
    • enero de 2007: XML Path Language 2.0, editada por Anders Berglund y otros
    • abril de 2014: XML Path Language (XPath) 3.0, editada por Jonathan Robie y otros
    • marzo de 2017: XML Path Language (XPath) 3.1, editada por Jonathan Robie y otros
  • XSLT (XSL Transformations)
    • noviembre de 1999: XSL Transformations 1.0, editada por James Clark
    • enero de 2007: XSL Transformations 2.0, editada por Michael Kay
    • junio de 2017: XSL Transformations 3.0, editada por Michael Kay
  • XSL/FO (XSL Formatting Objects)
    • octubre de 2001: Extensible Stylesheet Language (XSL) 1.0, editada por Sharon Adler y otros.
    • diciembre de 2006: Extensible Stylesheet Language (XSL) 1.1, editada por Anders Berglund

Actualmente (abril de 2018), el grupo de trabajo XSL mantiene las siguientes recomendaciones en fase de elaboración, pero hace años que no ya no trabaja nadie en ellas. En el mejor de los casos, puede que se publiquen como Notas del grupo de trabajo cuando se cierre definitivamente el grupo de trabajo.

  • Extensible Stylesheet Language (XSL) 2.0, editada por Dave Pawson, en estado de borrador desde 2012
  • Extensible Stylesheet Language (XSL) Requirements Version 2.0, en estado de borrador desde 2008

XQuery (XML Query)

El objetivo de XQuery es proporcionar herramientas para realizar consultas sobre documentos XML.

Grupo de trabajo XQuery en el W3C: Homepage - Plan de trabajo (2015-2017) (extendido hasta abril de 2018) - Plan de trabajo (2013-2015)

Actualmente (abril de 2018) el grupo XQuery no está cerrado, pero no se plantea publicar nuevas recomendaciones.

El grupo de trabajo XQuery ha publicado las siguientes recomendaciones:

  • diciembre de 2010: XQuery 1.0: An XML Query Language (2º edición), editada por Scott Boag y otros
  • diciembre de 2010: XML Syntax for XQuery 1.0 (XQueryX) (2º edición), editada por Jim Melton y Subramanian Muralidhar
  • marzo de 2011: XQuery Update Facility 1.0, editada por Jonathan Robie y otros
  • abril de 2014: XQuery 3.0: An XML Query Language, editada por Jonathan Robie y otros
  • abril de 2014: XQueryX 3.0, editada por Jim Melton
  • marzo de 2017: XQuery 3.1: An XML Query Language, editada por Jonathan Robie y otros
  • marzo de 2017: XQueryX 3.1, editada por Jim Melton y Josh Spiegel

y las siguientes notas de trabajo:

  • enero de 2011: XQuery Update Facility 1.0 Requirements, editada por Don Chamberlin y Jonathan Robie
  • enero de 2011: XQuery Update Facility 1.0 Use Cases, editada por Ioana Manolescu y Jonathan Robie
  • abril de 2014: XML Query 3.0 Use Cases, editada por Jonathan Robie y Tim Kraska
  • septiembre de 2014: XQuery Scripting Extension 1.0, editada por John Snelson y otros
  • diciembre de 2016: XQuery 3.1 Requirements and Use Cases, editada por Jonathan Robie
  • enero de 2017: XQuery Update Facility 3.0 , editada por John Snelson y Jim Melton
  • junio de 2017: XQuery Scripting Extension 1.0 Requirements, editada por Daniel Engovatov y otros
  • junio de 2017: XQuery Scripting Extension 1.0 Use Cases, editada por John Snelson

Nota: Después de XQuery 1.0 se publicó XQuery 3.0, es decir, no existen ni XQuery 1.1 ni XQuery 2.0. Probablemente la razón es utilizar el mismo número de versión que XPath 3.0, publicado en la misma fecha (abril de 2014).

Efficient XML Interchange (EXI)

Grupo de trabajo Efficient XML Interchange en el W3C: Homepage - Plan de trabajo (2015-2017) (extendido hasta diciembre de 2017) - Plan de trabajo (2013-2015) - Recomendaciones

El grupo de trabajo EXI no está cerrado, pero ha superado su fecha límite de trabajo (prolongada hasta diciembre de 2017). La intención del grupo era aprobar al menos la recomendación Canonical EXI. Actualmente (abril de 2018), el grupo sigue trabajando en dicha recomendación (y en tests relacionados con ella), por lo que no se puede descartar que la recomendación se acabe aprobando.

El grupo de trabajo Efficient XML Interchange ha publicado las siguientes recomendaciones:

  • febrero de 2014: Efficient XML Interchange (EXI) Format 1.0 (2º edición), editada por John Schneider y otros
  • septiembre de 2014: Efficient XML Interchange (EXI) Profile for limiting usage of dynamic memory, editada por Youenn Fablet y Daniel Peintner

Actualmente (abril de 2018), el grupo de trabajo Efficient XML Interchange está elaborando las siguientes recomendaciones:

  • Efficient XML Interchange (EXI) for JSON, editada por Daniel Peintner y Don Brutzman
  • Canonical EXI, editada por Sebastian Käbisch y Daniel Peintner
  • Efficient XML Interchange (EXI) Primer, editada por Daniel Peintner y Santiago Pericas-Geertsen
  • Efficient XML Interchange (EXI) Impacts, editada por Jaakko Kangasharju, en estado de borrador desde 2008
  • Efficient XML Interchange (EXI) Best Practices, editada por Mike Cokus y Daniel Vogelheim, en estado de borrador desde 2007
  • Efficient XML Interchange Evaluation, editada por Carine Bournez, en estado de borrador desde 2009
  • Efficient XML Interchange Measurements Note, editada por Greg White y otros, en estado de borrador desde 2007
